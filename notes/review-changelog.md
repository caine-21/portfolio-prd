# Review Changelog — v1.0 → v1.1

## 严重问题修复（3项）

### [FIX-1] 删除内部实验术语
- **原文：** 附录"接地性评分 | ... 类比 Day 5 实验的 Grounding Score"
- **问题：** "Day 5 实验"是个人研究术语，对外 PRD 的任何读者（面试官/工程师/业务方）都无法理解
- **修改：** 改为对外语言："采用 LLM-as-judge 方式评估，answer-aware 设计（综合考虑问题、检索片段和答案三者关系）"；技术实现细节移至 interview-defense.md

### [FIX-2] 补充多租户知识库隔离
- **原文：** F3 及边界条件完全未提多租户场景
- **问题：** B 端 SaaS 基础需求，不同客户数据必须物理隔离；面试官若有 B 端经验必然追问
- **修改：**
  - F3 加入"多租户隔离"段落（独立向量索引，workspace_id 限定检索范围）
  - 验收标准加入：跨租户请求被拒绝（403）并写入审计日志
  - 边界条件加入：跨租户请求触发 403 + 审计日志
  - 非目标加入："跨租户知识库共享"明确排除
  - 术语附录加入"多租户隔离"词条

### [FIX-3] 修正"RAG 引用覆盖率"指标定义
- **原文：** KR "输出内容可追溯至来源（RAG 引用标注），引用覆盖率 ≥ 90%"
- **问题：** 分母不明确——"所有生成"还是"知识库命中的生成"？两种定义目标完全不同；若知识库未命中时也要求标注，则该指标强制要求 100% 命中，逻辑矛盾
- **修改：** 改为"知识库命中时引用标注率 = 100%"，明确分母为命中条件下的生成；未命中时标记低置信度为单独机制

## 中等问题修复（4项）

### [FIX-4] 为"直接可用率 75%"加基线说明
- **原文：** KR "直接可用率 ≥ 75%"无任何说明
- **修改：** 在目标下方加 blockquote 说明：当前人工创作直接发布率约 50–60%（行业估算基线），AI 辅助目标提升 15–25pp；未达标触发模板优化流程

### [FIX-5] 明确 F6 评分的 evaluator 类型
- **原文：** "接地性评分：内容与知识库来源的语义相关度"（措辞模糊）
- **问题：** 工程师会问"是余弦相似度还是 LLM-judge？"——两种实现差异极大
- **修改：** F6 评分表格明确写"LLM-as-judge：基于检索到的上下文片段，评估答案是否准确反映来源内容"；加注 answer-aware 设计说明

### [FIX-6] 修正"Pearson ≥ 0.6"悬空数字
- **原文：** 验收标准"评分与人工判断相关性 ≥ 0.6（Pearson）"作为验收条件
- **问题：** Portfolio 中没有对应 Pearson 相关性实验；此数字无法在上线前验证
- **修改：** 改为"上线后通过人工标注数据验证，目标 Pearson ≥ 0.6"，放入 F6 评分说明，而非功能验收硬标准

### [FIX-7] 补充发布后撤回机制
- **原文：** 风险降级只写预防（强制审核），无应急响应
- **修改：**
  - F5 审核功能加入"发布后撤回"：已发布内容支持一键撤回，版本历史保留 90 天
  - 验收标准加入：已发布内容可一键撤回，操作写入审计日志（P0）
  - 风险表 RAG 幻觉条目加入"发布后支持一键撤回；异常内容保留 90 天供审计"
